## Base URL

Make all API requests to:

```
https://api.screenshothis.com
```

## Authentication

You'll authenticate using API keys passed as query parameters. Create your API key in the [dashboard](https://screenshothis.com/keys).

### API key format

Choose the appropriate key type for your environment:

- **Test keys**: Start with `ss_test_` for development and testing
- **Live keys**: Start with `ss_live_` for production use

### How to authenticate

Add your API key as a query parameter in every request:

<Tabs>
<Tab title="cURL">
```bash
curl "https://api.screenshothis.com/v1/screenshots/take?api_key=ss_test_1234567890abcdef&url=https://example.com"
```
</Tab>

<Tab title="JavaScript">
```typescript
const response = await fetch(
  `https://api.screenshothis.com/v1/screenshots/take?api_key=${apiKey}&url=https://example.com`
);
```
</Tab>

<Tab title="Python">
```python
import requests

response = requests.get(
    'https://api.screenshothis.com/v1/screenshots/take',
    params={
        'api_key': 'ss_test_1234567890abcdef',
        'url': 'https://example.com'
    }
)
```
</Tab>
</Tabs>

<Info>
**Client-side security**: You can safely use API keys in client-side applications thanks to domain whitelisting. Configure allowed domains in your [dashboard](https://screenshothis.com/keys) to restrict where your API key can be used.
</Info>

## Rate limits

We enforce rate limiting per API key to ensure fair usage and optimal performance:

| Plan | Requests per minute | Screenshots per month |
|------|--------------------|-----------------------|
| **Free** | 10 | 100 |
| **Pro** | 60 | 1,000+ |
| **Enterprise** | Custom | Custom |

### Rate limit headers

Check your current usage with these response headers:

```
X-RateLimit-Limit: 10
X-RateLimit-Remaining: 9
X-RateLimit-Reset: 1640995200
```

<Tip>
Monitor these headers to implement proper retry logic and avoid hitting rate limits.
</Tip>

## Error handling

The API uses standard HTTP status codes and returns structured JSON error responses.

### Error response format

When an error occurs, you'll receive a JSON response with error details:

```json
{
  "error": "Invalid API key provided",
  "requestId": "req_1234567890abcdef"
}
```

### HTTP status codes

| Status Code | Meaning | Example Response |
|-------------|---------|------------------|
| `200` | Success | Screenshot image binary data |
| `400` | Bad Request | Invalid parameters or malformed URL |
| `403` | Forbidden | Invalid API key or quota exceeded |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server-side error occurred |

## Response formats

### Image response (default)

By default, you'll receive the screenshot image directly as binary data:

<RequestExample>
```bash cURL
curl "https://api.screenshothis.com/v1/screenshots/take?api_key=ss_test_1234567890abcdef&url=https://example.com" \
  -o screenshot.jpeg
```
</RequestExample>

<ResponseExample>
```
HTTP/1.1 200 OK
Content-Type: image/jpeg
Content-Length: 245760
Cache-Control: public, max-age=3600

[Binary image data]
```
</ResponseExample>

The `Content-Type` header will match your requested format:
- `image/jpeg` for JPEG format
- `image/png` for PNG format
- `image/webp` for WebP format
