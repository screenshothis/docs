# Get Started with Screenshothis

Ready to start capturing beautiful screenshots? Follow this guide to make your first API call in under 5 minutes.

<Steps>
  <Step title="Create Your Account">
    Sign up for a free account at [screenshothis.com](https://screenshothis.com/register) to get started with 100 free screenshots per month.
  </Step>
  <Step title="Create Your API Key">
    Navigate to [API Keys](https://screenshothis.com/keys) and create your first API key. It will start with `ss_test_` for development or `ss_live_` for production.
  </Step>
  <Step title="Make Your First Request">
    Use the API key to capture your first screenshot using any of the methods below.
  </Step>
</Steps>

## Your First Screenshot

<Tabs>
  <Tab title="cURL">
    The simplest way to test the API:

    ```bash
    curl -X GET "https://api.screenshothis.com/v1/screenshots/take?api_key=ss_test_your_api_key_here&url=https://github.com" \
      -o screenshot.jpeg
    ```

    <Check>Your screenshot will be saved as `screenshot.jpeg`</Check>
  </Tab>

  <Tab title="JavaScript">
    Perfect for web applications and Node.js:

    ```bash
    npm install @screenshothis/sdk
    ```

    ```typescript
    import { Screenshothis } from "@screenshothis/sdk";

    const screenshothis = new Screenshothis();

    const imageData = await screenshothis.screenshots.take({
      apiKey: "ss_test_your_api_key_here",
      url: "https://github.com"
    });

    // Save to file (Node.js)
    const fs = require('fs');
    fs.writeFileSync('screenshot.jpeg', Buffer.from(imageData));
    ```

    <Card title="Complete SDK Guide" icon="arrow-right" href="/sdk-js">
      For comprehensive documentation including error handling, tree-shaking, React/Vue examples, and all available options
    </Card>
  </Tab>

  <Tab title="Python">
    Ideal for data science and backend services:

    ```python
    import requests

    response = requests.get(
        'https://api.screenshothis.com/v1/screenshots/take',
        params={
            'api_key': 'ss_test_your_api_key_here',
            'url': 'https://github.com'
        }
    )

    with open('screenshot.jpeg', 'wb') as f:
        f.write(response.content)
    ```

    <Check>Screenshot saved to `screenshot.jpeg`</Check>
  </Tab>

  <Tab title="PHP">
    Great for WordPress and Laravel integrations:

    ```php
    <?php
    $params = http_build_query([
        'api_key' => 'ss_test_your_api_key_here',
        'url' => 'https://github.com'
    ]);

    $url = 'https://api.screenshothis.com/v1/screenshots/take?' . $params;
    $screenshot = file_get_contents($url);
    file_put_contents('screenshot.jpeg', $screenshot);
    ?>
    ```
  </Tab>
</Tabs>

## API Parameters

| Parameter | Description | Default | Options |
|-----------|-------------|---------|---------|
| `url` | Website URL to screenshot | **Required** | Any valid URL |
| `api_key` | Your API key | **Required** | `ss_test_` or `ss_live_` |
| `width` | Viewport width | `1920` | Pixel value |
| `height` | Viewport height | `1080` | Pixel value |
| `format` | Image format | `jpeg` | `jpeg`, `png`, `webp` |
| `quality` | Image quality (20-100) | `80` | `20` to `100` |
| `is_mobile` | Mobile device simulation | `false` | `true`, `false` |
| `block_ads` | Block advertisements | `false` | `true`, `false` |
| `prefers_color_scheme` | Color scheme preference | `light` | `light`, `dark` |

<Note>
**Built-in Limits**: Screenshothis enforces an 8KB header size limit and 4KB cookie size limit for security and performance.
</Note>

## Real-World Use Cases

<AccordionGroup>
  <Accordion title="Social Media Previews" icon="share">
    Generate dynamic `og:image` tags for better social media sharing:

        ```typescript
    import { Screenshothis } from "@screenshothis/sdk";

    // Generate preview for blog post
    const generatePreview = async (blogPostUrl: string, apiKey: string) => {
      const screenshothis = new Screenshothis();

      const imageData = await screenshothis.screenshots.take({
        apiKey,
        url: blogPostUrl,
        width: 1200,
        height: 630,
        format: "jpeg"
      });

      // Convert to base64 for direct use in HTML
      const base64 = Buffer.from(imageData).toString('base64');
      return `data:image/jpeg;base64,${base64}`;
    };

    // Usage in HTML
    const previewUrl = await generatePreview("https://myblog.com/post", apiKey);
    // <meta property="og:image" content="data:image/jpeg;base64,..." />
    ```

    <Check>Perfect for Twitter, LinkedIn, and Facebook sharing</Check>

  <Accordion title="Error Handling with SDK" icon="shield">
    Robust error handling using the TypeScript SDK:

    ```typescript
    import { Screenshothis } from "@screenshothis/sdk";
    import * as errors from "@screenshothis/sdk/models/errors";

    const screenshothis = new Screenshothis();

    try {
      const imageData = await screenshothis.screenshots.take({
        apiKey: "ss_test_your_api_key_here",
        url: "https://example.com"
      });

      // Save the screenshot
      const fs = require('fs');
      fs.writeFileSync('screenshot.jpeg', Buffer.from(imageData));
      console.log('Screenshot saved successfully!');
    } catch (error) {
      if (error instanceof errors.ScreenshothisError) {
        console.log(error.message);
        console.log(error.statusCode);

        if (error instanceof errors.ForbiddenError) {
          console.log(error.data$.error);
          console.log(error.data$.requestId);
        }
      }
    }
    ```

    <Check>TypeScript provides full type safety for error handling</Check>
  </Accordion>

  <Accordion title="Browser Integration" icon="globe">
    Using screenshots in web applications:

    ```typescript
    import { Screenshothis } from "@screenshothis/sdk";

    const screenshothis = new Screenshothis();

    // Create a screenshot preview function
    const createPreview = async (url: string, apiKey: string) => {
      try {
        const imageData = await screenshothis.screenshots.take({
          apiKey,
          url,
          width: 800,
          height: 600,
          format: "webp"
        });

        // Create blob URL for immediate display
        const blob = new Blob([imageData], { type: 'image/webp' });
        return URL.createObjectURL(blob);
      } catch (error) {
        console.error('Failed to create screenshot:', error);
        return null;
      }
    };

    // Usage in React/Vue/etc
    const handleScreenshot = async () => {
      const previewUrl = await createPreview(inputUrl, apiKey);
      if (previewUrl) {
        setImageSrc(previewUrl);
      }
    };
    ```

    <Tip>Use WebP format for smaller file sizes in browsers</Tip>
  </Accordion>

    <Accordion title="Direct URL Rendering (No API Call Needed!)" icon="link">
    **The simplest approach - construct the URL directly:**

            ```typescript
    import type { GetV1ScreenshotsTakeRequest } from "@screenshothis/sdk/models/operations";

    // Use SDK's existing type, excluding required fields that are handled separately
    type ScreenshotOptions = Omit<GetV1ScreenshotsTakeRequest, "apiKey" | "url">;

    // Pure URL construction - no SDK call needed!
    const buildScreenshotUrl = (
      url: string,
      apiKey: string,
      options?: ScreenshotOptions
    ): string => {
      const params = new URLSearchParams({
        api_key: apiKey,
        url: url,
        ...options // All properly typed now!
      });

      return `https://api.screenshothis.com/v1/screenshots/take?${params}`;
    };

    // Usage - instant, no async needed!
    const screenshotUrl = buildScreenshotUrl("https://github.com", apiKey, {
      format: "png",
      width: 800,
      height: 600
    });

    // Use directly in HTML
    document.getElementById('myImage').src = screenshotUrl;
    // Or: <img src={screenshotUrl} alt="Screenshot" />
    ```

            **React/Vue Component (Zero Loading State!):**
    ```tsx
    import type { GetV1ScreenshotsTakeRequest } from "@screenshothis/sdk/models/operations";

    type ScreenshotOptions = Omit<GetV1ScreenshotsTakeRequest, "apiKey" | "url">;

    interface ScreenshotImageProps {
      url: string;
      apiKey: string;
      options?: ScreenshotOptions;
    }

    const ScreenshotImage = ({ url, apiKey, options }: ScreenshotImageProps) => {
      const screenshotUrl = buildScreenshotUrl(url, apiKey, {
        format: "webp",
        width: 800,
        height: 600,
        ...options
      });

      return <img src={screenshotUrl} alt="Screenshot" loading="lazy" />;
    };

    // Usage - all SDK types available with IntelliSense!
    <ScreenshotImage
      url="https://example.com"
      apiKey="ss_test_your_key"
      options={{
        width: 1200,
        height: 630,
        blockAds: true,
        prefersColorScheme: "dark",
        selector: "#main"
      }}
    />
    ```

        **Advanced Options (Using SDK Types!):**
    ```typescript
    const screenshotUrl = buildScreenshotUrl("https://github.com", apiKey, {
      // All options from GetV1ScreenshotsTakeRequest are available

    ```
